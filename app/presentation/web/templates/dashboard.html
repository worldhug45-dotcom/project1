Stage 10 Rules

Current Boundaries

  • /health is queried separately and only powers the read-only badge at the top of the dashboard.
  • /api/keywords keeps powering the shared keyword snapshot, while /api/keywords/core, /api/keywords/supporting, and /api/keywords/exclude each save only their own keyword group changes.
  • Collect, export, and observe are controllable from the web shell in this stage.
  • The same manual_run.py collect path is reused instead of introducing a new engine.
  • The same manual_run.py export path is reused for export.
  • The same manual_run.py observe path is reused for observation runs.
  • Core, supporting, and exclude keywords can all be edited from their own panels in the dashboard.
  • If the same action is already active, the dashboard returns the current running state instead of starting another one.
