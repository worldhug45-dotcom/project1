# 키워드 보정 분석 메모

## 목적

이 문서는 기업마당 실제 API diagnostics 결과를 바탕으로 `saved_count`를 개선하기 위한 키워드 보정 근거를 정리한다.

## 분석 기준

- 실제 기업마당 API 응답 사용
- 동일한 `BizinfoNoticeRaw -> BizinfoNoticeNormalizer -> evaluate_keywords` 흐름 기준
- 진단 출력(`collect_diagnostic`)과 추가 시뮬레이션 결과를 함께 사용

## 1차 진단 결과 요약

실제 API 상위 20건 진단에서 주요 skipped 사유는 아래와 같았다.

- `no_keyword_match`: 12건
- `excluded_keyword`: 7건
- `normalization_value_error`: 1건

이후 자유 형식 날짜를 선택 값 `None`으로 정규화하도록 보강한 뒤, `normalization_value_error` 1건은 저장 가능 상태로 전환되었다.

## no_keyword_match 세부 관찰

실제 `no_keyword_match` 공고 중 기술 범위와 가까운 사례:

1. `[경기] 포천시 2026년 사물인터넷(IoT) 측정기기 부착 지원사업 재공고`
2. `[경남] 2026년 비제조산업 점프업 기업 지원사업 참여기업 추가 모집 공고`
   - 요약에 `ICT/SW` 포함
3. `[경기] 안산시 2026년 강소기업 육성 지원사업(3단계) 신규지원 대상과제 공고`
   - 요약에 `SW` 포함
4. `[경북] 경주시 2026년 2차 스마트팜 육성지원사업 시행 공고`
   - 요약에 `ICT` 포함

반면 아래와 같은 사례는 현재 범위를 넓혀도 잡음 가능성이 높아 보였다.

- `IP SOLUTION`
- `디자인 프로젝트`
- `시스템반도체`
- 일반 `강소기업`, `보험료`, `혁신제품`, `배출권거래제`

## 후보 키워드 시뮬레이션

실제 API 상위 100건을 기준으로 간단 시뮬레이션을 수행했다.

### 기준안

- 핵심: 기존 유지
- 보조: 기존 유지
- 제외: 기존 유지

결과:

- `eligible = 10`
- `skipped = 90`

### 후보안 A

- 보조 추가: `사물인터넷`, `IoT`
- 도메인 매핑: `infra`

결과:

- `eligible = 11`
- `skipped = 89`

### 후보안 B

- 보조 추가: `사물인터넷`, `IoT`, `ICT`, `SW`, `소프트웨어`
- 도메인 매핑:
  - `사물인터넷`, `IoT` -> `infra`
  - `ICT`, `SW`, `소프트웨어` -> `si`

결과:

- `eligible = 15`
- `skipped = 85`

후보안 B가 가장 작은 범위 변경으로 저장 가능 건수를 늘렸다.

## 최종 보정안

### 핵심 키워드

- 변경 없음

### 보조 키워드

- 추가
  - `사물인터넷`
  - `IoT`
  - `ICT`
  - `SW`
  - `소프트웨어`

### 강제 제외 키워드

- 변경 없음

현재 라운드에서는 `excluded_keyword`보다 `no_keyword_match`가 더 큰 병목이었고, 제외 키워드를 섣불리 완화하면 잡음 공고가 함께 늘어날 위험이 있어 유지한다.

### 도메인 매핑

- `사물인터넷` -> `infra`
- `IoT` -> `infra`
- `ICT` -> `si`
- `SW` -> `si`
- `소프트웨어` -> `si`

## 설정 반영안

`config/settings.example.toml`의 `keywords.supporting`에 아래 값을 추가한다.

```toml
supporting = [
  "데이터",
  "빅데이터",
  "클라우드",
  "인프라",
  "서버",
  "네트워크",
  "사물인터넷",
  "IoT",
  "보안",
  "ICT",
  "SW",
  "소프트웨어",
  "IT서비스",
  "유지보수",
]
```

동시에 코드의 도메인 매핑 테이블에도 같은 키워드를 반영해야 한다.

## 기대 효과

- 실제 API 상위 20건 기준:
  - 기존 `saved_count = 1`
  - 보정 후 실제 수동 검증 결과 `saved_count = 4`
- 실제 API 상위 100건 기준:
  - 기존 `eligible = 10`
  - 보정 후 `eligible = 15`

## 보류 항목

아래 키워드는 잡음 가능성이 높아 이번 라운드에서는 반영하지 않는다.

- `플랫폼`
- `IP`
- `SOLUTION`
- `로봇`
- `반도체`
- `디자인`
