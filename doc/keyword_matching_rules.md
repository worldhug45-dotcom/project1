# 키워드 판정 규칙 정의

## 1. 목적

이 문서는 1차 MVP에서 공고가 AI, 인프라, SI 사업기회에 해당하는지 판정하는 키워드 규칙을 정의한다.  
판정 결과는 `Notice.business_domains`, `Notice.primary_domain`, `Notice.match_keywords` 생성 기준으로 사용한다.

## 2. 적용 범위

- 포함: 매칭 대상 필드, 전처리 규칙, 키워드 그룹 우선순위, 도메인 매핑, 대표 도메인 선정, 적격 판정 기준
- 제외: 외부 API 호출 구현, DB 저장 구현, 엑셀 출력 구현, 로그 구현

## 3. 기본 원칙

- 키워드 판정과 도메인 분류는 `Domain` 계층의 비즈니스 규칙이다.
- 키워드 목록은 `settings_schema.md`의 `keywords` 섹션에서 읽는다.
- 코드에는 키워드 목록을 하드코딩하지 않는다.
- 이 문서의 도메인 매핑과 우선순위는 기본 판정 규칙이다.
- 강제 제외 키워드는 핵심/보조 키워드보다 항상 우선한다.
- 적격 판정된 공고만 저장 및 엑셀 산출 대상이 된다.

## 4. 매칭 대상 필드

키워드 매칭은 내부 공통 모델 기준으로 아래 필드를 대상으로 한다.

| 필드 | 사용 여부 | 설명 |
| --- | --- | --- |
| `title` | 사용 | 가장 중요한 매칭 대상이다. |
| `organization` | 사용 | 기관명에 사업 영역 단서가 포함될 수 있어 보조적으로 사용한다. |
| `summary` | 사용 | 원천에서 제공하는 설명 또는 요약이 있을 때 사용한다. |
| `raw_source_name` | 사용 | 원천 서비스명 또는 세부 출처명에 도메인 단서가 있을 때 사용한다. |
| `url` | 미사용 | URL 문자열은 사업 영역 판단 대상에서 제외한다. |
| `posted_at`, `end_at`, `collected_at` | 미사용 | 날짜 값은 키워드 판정 대상이 아니다. |

원천 API에 별도 상세 설명 필드가 있으면 정규화 단계에서 `summary`로 전달한 뒤 매칭한다.  
AI 요약 생성은 1차 MVP 범위가 아니므로 `summary`는 원천 제공 텍스트만 사용한다.

## 5. 전처리 규칙

매칭 전에 대상 텍스트와 키워드에 동일한 전처리를 적용한다.

1. HTML 태그와 HTML entity를 제거 또는 디코딩한다.
2. Unicode 정규화는 `NFKC`를 사용한다.
3. 앞뒤 공백을 제거한다.
4. 연속 공백은 하나의 공백으로 축약한다.
5. 영문자는 대소문자를 구분하지 않도록 casefold 처리한다.
6. 한글, 숫자, 영문 의미는 보존한다.
7. 쉼표, 괄호, 줄바꿈, 탭 등 구분자는 공백처럼 취급할 수 있다.
8. 형태소 분석, 유의어 확장, AI 기반 의미 판정은 1차 MVP에서 사용하지 않는다.

## 6. 짧은 영문 키워드 처리

`AI`, `DX`, `SI`처럼 짧은 영문 키워드는 단순 부분 문자열로 매칭하지 않는다.  
예를 들어 `mail`, `basis` 같은 단어 내부에 포함된 문자열은 매칭하지 않는다.

짧은 영문 키워드는 아래 조건을 만족할 때 매칭한다.

- 앞뒤에 영문자 또는 숫자가 직접 붙어 있지 않다.
- 한글, 공백, 구두점과 붙어 있는 경우는 매칭한다.

예시:

| 텍스트 | 키워드 | 결과 | 이유 |
| --- | --- | --- | --- |
| `AI 기반 서비스` | `AI` | 매칭 | 공백과 붙어 있다. |
| `AI사업화 지원` | `AI` | 매칭 | 한글과 붙어 있다. |
| `SI 구축 용역` | `SI` | 매칭 | 공백과 붙어 있다. |
| `mail 시스템` | `AI` | 미매칭 | 영문 단어 내부 문자열이다. |
| `basis data` | `SI` | 미매칭 | 영문 단어 내부 문자열이다. |

## 7. 키워드 그룹 우선순위

키워드는 아래 우선순위로 판정한다.

1. 강제 제외 키워드
2. 핵심 키워드
3. 보조 키워드

강제 제외 키워드가 하나라도 매칭되면 핵심 또는 보조 키워드가 함께 매칭되어도 부적격으로 판정한다.  
부적격 공고는 저장 및 엑셀 산출 대상에서 제외한다.

## 9. business_domains 생성 규칙

`business_domains`는 매칭된 핵심/보조 키워드의 기본 도메인을 중복 없이 모아 생성한다.

- 강제 제외 키워드는 `business_domains`를 생성하지 않는다.
- 같은 도메인의 키워드가 여러 개 매칭되어도 도메인은 한 번만 포함한다.
- 도메인 순서는 `primary_domain` 선정 우선순위에 맞춰 정렬한다.
- 적격 공고의 `business_domains`는 비어 있으면 안 된다.

## 10. primary_domain 선정 규칙

`primary_domain`은 `business_domains` 중 대표 도메인 1개이다.

선정 우선순위는 아래와 같다.

1. 핵심 키워드에서 매칭된 도메인
2. 보조 키워드에서 매칭된 도메인
3. 도메인 우선순위
4. 매칭 대상 필드 우선순위
5. 텍스트 내 최초 출현 순서

도메인 우선순위는 아래와 같다.

1. `ai`
2. `dx`
3. `si`
4. `infra`
5. `data`
6. `security`
7. `maintenance`

매칭 대상 필드 우선순위는 아래와 같다.

1. `title`
2. `summary`
3. `organization`
4. `raw_source_name`

예를 들어 `title`에서 `클라우드`가 매칭되고 `summary`에서 `AI`가 매칭되면 핵심 키워드인 `AI`의 도메인 `ai`를 `primary_domain`으로 선정한다.  
핵심 키워드가 없고 `title`에서 `보안`, `summary`에서 `데이터`가 매칭되면 도메인 우선순위에 따라 `data`를 `primary_domain`으로 선정한다.

## 11. match_keywords 생성 규칙

`match_keywords`는 매칭된 핵심/보조 키워드를 `MatchedKeyword` 목록으로 보존한다.

| 필드 | 생성 규칙 |
| --- | --- |
| `keyword` | 설정에 정의된 원래 키워드 표기를 사용한다. |
| `group` | `core` 또는 `supporting`을 기록한다. |
| `domain` | 도메인 매핑표의 기본 도메인을 기록한다. |

강제 제외 키워드가 매칭된 부적격 공고는 최종 `Notice` 저장 대상이 아니므로 `Notice.match_keywords` 생성 대상이 아니다.  
다만 판정 결과나 진단 정보에서는 제외 키워드를 별도로 기록할 수 있다.

## 12. 적격 판정 기준

공고는 아래 조건을 모두 만족할 때 적격으로 판정한다.

- 강제 제외 키워드가 매칭되지 않는다.
- 핵심 키워드가 1개 이상 매칭되거나, 보조 키워드가 1개 이상 매칭된다.
- 매칭된 핵심/보조 키워드에서 `business_domains`를 1개 이상 생성할 수 있다.
- `primary_domain`을 1개 선정할 수 있다.

아래 조건 중 하나라도 해당하면 부적격으로 판정한다.

- 강제 제외 키워드가 1개 이상 매칭된다.
- 핵심/보조 키워드가 하나도 매칭되지 않는다.
- 매칭 키워드가 있지만 도메인 매핑을 찾을 수 없다.
- `business_domains` 또는 `primary_domain`을 생성할 수 없다.

## 13. 판정 예시

| title | summary | 매칭 | 판정 | primary_domain |
| --- | --- | --- | --- | --- |
| `AI 기반 업무 자동화 지원사업` | 없음 | `AI` | 적격 | `ai` |
| `클라우드 인프라 고도화 용역` | 없음 | `클라우드`, `인프라` | 적격 | `infra` |
| `정보화 시스템 통합 유지보수` | 없음 | `정보화`, `시스템 통합`, `유지보수` | 적격 | `si` |
| `AI 경진대회 참가자 모집` | 없음 | `AI`, `경진대회` | 부적격 | 없음 |
| `문화행사 운영 용역` | 없음 | `문화`, `행사` | 부적격 | 없음 |

## 14. 설정과의 관계

- 실제 키워드 목록은 `config/settings.toml`의 `keywords` 섹션에서 읽는다.
- 이 문서의 기본 키워드와 도메인 매핑은 1차 MVP의 초기 기준이다.
- 설정 파일에서 키워드를 추가할 경우 도메인 매핑도 함께 정의해야 한다.
- 도메인 매핑이 없는 키워드는 적격 판정에 사용할 수 없다.

## 15. 1차 확정 사항

- 매칭 대상 필드는 `title`, `organization`, `summary`, `raw_source_name`으로 한다.
- URL과 날짜 값은 키워드 매칭 대상에서 제외한다.
- 전처리는 `NFKC`, 공백 정규화, 영문 casefold를 기본으로 한다.
- 짧은 영문 키워드는 영문자/숫자 내부 부분 문자열로 매칭하지 않는다.
- 강제 제외 키워드는 핵심/보조 키워드보다 우선한다.
- 보조 키워드만 매칭되어도 적격으로 판정할 수 있다.
- `business_domains`는 매칭된 핵심/보조 키워드의 도메인 집합이다.
- `primary_domain`은 핵심 여부, 도메인 우선순위, 필드 우선순위, 최초 출현 순서로 선정한다.

## 16. 남은 결정 사항

- 설정 파일에서 도메인 매핑을 표현하는 구체적 구조
- 제외 키워드 진단 정보를 로그에 기록하는 방식
- 키워드별 가중치 또는 점수화 도입 여부
