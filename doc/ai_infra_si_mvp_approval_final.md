# AI·인프라·SI 사업기회 탐지용 공고 수집 시스템 1차 MVP 구축 승인 요청안

## 1. 승인 요청 개요
본 안건은 당사의 AI, 인프라, SI 사업영역과 관련된 외부 사업기회를 체계적으로 탐지하기 위해 공고 수집 시스템의 1차 MVP 구축을 승인받기 위한 것이다.  
1차 MVP는 **2트랙 구조**로 추진한다.

- **Track 1:** 기업마당 기반 지원사업 공고 수집
- **Track 2:** 나라장터 기반 입찰공고 수집

기업마당은 **`지원사업정보 API`**를 통해 기관별·분야별 최신 지원사업 공고 정보를 제공한다.  
나라장터는 물품·용역·공사·외자 입찰공고 목록, 상세, 변경이력 등을 제공하는 입찰공고정보서비스를 제공하며, 조달청 공지에 따르면 해당 서비스의 **25개 오퍼레이션 전체는 2025년 2월 4일 13시부터 24시간 개방**됐다.

## 2. 추진 목적
본 시스템의 1차 목적은 다음과 같다.

- AI, 인프라, SI와 관련된 사업기회를 정기적으로 수집할 것
- 지원사업 공고와 조달 입찰공고를 분리해 관리할 것
- 공통 기준으로 정규화해 저장할 것
- 검토 가능한 엑셀 산출물을 자동 생성할 것
- 반복 실행과 장애 확인이 가능한 운영 기반을 마련할 것

## 3. 1차 승인 범위
1차 승인 범위는 아래로 한정한다.

- 기업마당 공고 조회
- 나라장터 공고 조회
- 키워드 기반 적격 여부 판정
- 공통 필드 정규화
- DB 저장
- 엑셀 산출물 생성
- 로그 생성

## 4. 본 승인 범위 제외 항목
아래 항목은 1차 범위에서 제외한다.

- NTIS 연동
- K-Startup 연동
- AI 요약
- Word 출력
- 알림 연동
- 검색 UI
- 우선순위 자동 점수화

NTIS는 국가R&D 통합공고 및 Open API 활용 축이 있어 2차 우선 검토 대상으로 둔다.  
K-Startup은 현재도 창업지원사업 공고 확인이 가능하나, 당사의 1차 사업기회 탐지 범위에서는 기업마당·나라장터 대비 후순위로 판단한다.

## 5. 1차 MVP 정의
1차 MVP는 다음과 같이 정의한다.

> **기업마당과 나라장터의 최근 공고를 조회하고, AI·인프라·SI 관련 적격 공고를 선별하여 공통 형식으로 저장한 뒤, 검토 가능한 엑셀 파일로 산출하는 최소 운영 가능 버전**

## 6. 완료 기준
완료 기준은 트랙별·단계별로 분리해 적용한다.

### 6-1. 조회 기준
- 기업마당 상위 20건 조회 성공
- 나라장터 상위 20건 조회 성공

### 6-2. 파싱 기준
- 조회 대상 기준 필수 필드 파싱 성공률 95% 이상

### 6-3. 저장 기준
- 키워드 규칙에 의해 적격으로 판정된 건은 100% 저장
- 동일 건 재수집 시 중복 저장률 0%

### 6-4. 산출물 기준
- 엑셀 생성 성공
- 로그 생성 성공

### 6-5. 운영 기준
- 검증 기간 5영업일
- 치명적 실패 0회
- 수동 재실행 성공률 100%

## 8. 데이터 분류 기준
공고 데이터는 아래 기준으로 분류한다.

- `notice_type`: `support / bid / rnd / startup`
- `business_domains`: 다중 선택 허용
- `primary_domain`: 대표 1개
- `match_keywords`: 매칭 키워드 목록

즉, 실제 저장은 다중 태깅을 허용하고, 보고용 분류는 대표 도메인 1개를 사용한다.

## 9. 산출물 기준
엑셀 산출물은 최소 2개 시트로 구성한다.

- `support_notices`: 기업마당 지원사업 공고
- `bid_notices`: 나라장터 입찰공고

필수 컬럼은 아래와 같다.

- `source`
- `notice_type`
- `primary_domain`
- `title`
- `organization`
- `posted_at`
- `end_at`
- `status`
- `url`
- `match_keywords`
- `collected_at`

엑셀은 날짜 형식을 `YYYY-MM-DD`로 통일하고, URL은 클릭 가능해야 하며, 사용자가 5분 이내 최근 공고를 검토할 수 있는 가독성을 확보해야 한다.

## 10. 장애 기준
### 10-1. 치명적 실패
아래 중 하나라도 발생하면 치명적 실패로 본다.

- 기업마당 또는 나라장터 중 한 트랙이라도 전체 수집 실패
- DB 저장 실패
- 엑셀 미생성
- 로그 미생성

### 10-2. 비치명적 오류
아래는 비치명적 오류로 본다.

- 일부 공고의 선택 필드 누락
- 재시도 후 복구된 오류
- 일부 공고 제외 후 전체 실행 성공

단, 외부 API 제공기관의 정기 점검 또는 공지된 일시 중단은 운영 로그에 별도 기록하고, 내부 장애와 구분해 관리한다.  
또한 외부 API 공지사항(운영시간 제한, 서비스 변경, 일시 장애)은 운영 검토 항목에 포함하며, 검증 기간 중 발생한 외부 이슈는 내부 품질 실패와 구분해 기록한다.

## 11. 일정
1차 일정은 아래를 기준으로 한다.

- M1 승인 및 기준 확정: 2일
- M2 데이터/키워드 기준 확정: 2일
- M3 기업마당 수집 구현: 3일
- M4 나라장터 수집 구현: 4일
- M5 저장/중복/엑셀 검증: 3일
- M6 운영 검증: 5영업일

총 소요 예상은 **약 4주 내외**로 본다.
